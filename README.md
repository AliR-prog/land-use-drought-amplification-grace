# Middle East Hydrological Drought Attribution Using GRACE and Explainable Machine Learning

This repository contains the Python workflow used to reproduce the main analysis in the manuscript:

**Land-Use-Driven Amplification of Hydrological Drought in the Middle East: Basin-Scale Attribution Using GRACE, Land Cover Change, and Explainable Machine Learning**

## Repository purpose
The code in this repository builds a basin-year panel dataset, trains a Random Forest regression model, computes SHAP-based feature attribution, and runs a counterfactual land-use experiment to estimate land-use-driven drought amplification.

## Repository structure
- `src/01_build_panel.py` : merges GRACE, climate, and land-use input tables into one basin-year dataset
- `src/02_train_rf.py` : trains a Random Forest regression model and reports performance
- `src/03_shap_counterfactual.py` : computes SHAP values and counterfactual amplification
- `src/quick_test.py` : runs a minimal end-to-end test using example input files
- `example_data/` : small toy datasets for testing the workflow
- `results/` : output tables generated by the scripts
- `requirements.txt` : Python packages needed to run the code
- `LICENSE` : open-source license
- `docs/COMPUTER_CODE_AVAILABILITY.txt` : manuscript text for the Computer Code Availability section

## Input data expected by the scripts
The scripts expect three CSV files:
1. `grace_basin_year.csv`
   - Columns: `BasinID`, `Year`, `TWSAmin_cm`
2. `climate_basin_year.csv`
   - Columns: `BasinID`, `Year`, `P`, `T`, `P_anom`, `T_anom`
3. `lulc_basin_year.csv`
   - Columns: `BasinID`, `Year`, `fUrban`, `fCropland`, `fVegetation`

You can replace the toy example files with the real basin-year tables 
## Installation
Create a Python environment and install the required packages:

```bash
pip install -r requirements.txt
```

## Quick test
Run the example workflow with the toy data:

```bash
python src/quick_test.py
```

This will:
1. build the panel dataset,
2. train the Random Forest model,
3. compute SHAP values,
4. run the counterfactual experiment,
5. save outputs to the `results/` folder.

## How to run the full workflow
### Step 1: Build the basin-year panel
```bash
python src/01_build_panel.py   --grace example_data/grace_basin_year.csv   --climate example_data/climate_basin_year.csv   --lulc example_data/lulc_basin_year.csv   --output results/panel.csv
```

### Step 2: Train the Random Forest model
```bash
python src/02_train_rf.py   --panel results/panel.csv   --output results/model_metrics.csv
```

### Step 3: SHAP analysis and counterfactual experiment
```bash
python src/03_shap_counterfactual.py   --panel results/panel.csv   --output-prefix results/final
```
